### **1️⃣ User Input / Target Definition**

**وظيفتها:** تحديد الهدف اللي هيتعمل عليه الـ recon و الـ testing.

**تفاصيل:**

- **Target Type:** لازم تحدد نوع الهدف عشان تختار الأدوات المناسبة.
    - **Domain:** لو عندك موقع أو نطاق، هتحتاج أدوات subdomain enumeration و web scanning.
    - **IP Range:** لو عندك شبكة، هتستخدم أدوات Nmap, Masscan, etc.
    - **Application:** لو الهدف تطبيق محدد، ممكن تحتاج dynamic scanning أو API testing.
- **Input Validation:** التأكد من صحة الإدخال (مثلاً صيغة IP صحية، domain موجود، وما إلى ذلك) قبل بدء أي فحص.

**AI Role:**

- LLM أو قواعد بسيطة تتحقق من صحة الإدخال وتفهم نوع الهدف وتجهز البيانات للمرحلة التالية.

---

### **2️⃣ Decision Engine (AI or Rules)**

**وظيفتها:** تحديد الأدوات المناسبة وخطة الفحص بناءً على الهدف.

**تفاصيل:**

- **Analyze Target Type:** فهم الهدف، هل هو web app, network, cloud؟.
- **Select Recon Tools:** بناءً على نوع الهدف:
    - **Domain:** Subdomain Enum → Port Scan → Vulnerability Scan.
    - **IP Range:** Nmap → Masscan → Vulnerability Check.
- **Parameter Optimization (Optional AI):** تعديل إعدادات الأدوات (threads, scan depth, payload type) لتحسين الأداء والدقة.

**AI Role:**

- LLM أو Rule-based system لتحديد الأدوات.
- ممكن Reinforcement Learning أو Bayesian Optimization لتحسين إعدادات الأدوات تلقائيًا.

---

### **3️⃣ Execution / Orchestration**

**وظيفتها:** تشغيل الأدوات اللي اختارها الـ Decision Engine وجمع النتائج.

**تفاصيل:**

- **Run Tools Sequentially or Parallel:** الأدوات ممكن تشتغل واحدة واحدة أو متوازية حسب الموارد.
- **Capture Tool Output:** حفظ كل النتائج من الأدوات المختلفة.
- **Normalize Results → JSON Unified Format:** تحويل كل النتائج لشكل موحد يسهل تحليله في المراحل التالية.

**AI Role:**

- Orchestrator: يقرر ترتيب تشغيل الأدوات.
- يمكن للـ AI مراقبة الأداء وإدارة الأخطاء.

---

### **4️⃣ Feedback / Adaptive Loop**

**وظيفتها:** التكيف مع النتائج الجديدة وتحسين خطة الفحص.

**تفاصيل:**

- **Analyze Results:** فحص النتائج المستخرجة لمعرفة الأنماط أو النقاط المهمة.
- **Detect New Findings:** مثلاً اكتشاف subdomains جديدة أو open ports غير متوقعة.
- **Update Execution Plan (Next Steps):** تعديل خطة الأدوات التالية أو إضافة أدوات جديدة بناءً على النتائج.

**AI Role:**

- LLM أو Graph-based AI يحدد الخطوة التالية.
- Reinforcement Learning ممكن يستخدم لتحسين الفلو مع الوقت.

---

### **5️⃣ Reporting / Dashboard**

**وظيفتها:** تجميع النتائج وعرضها بشكل مفهوم.

**تفاصيل:**

- **Save Full_data.json:** حفظ كل البيانات المجمعة في ملف واحد.
- **Optional: Generate HTML/PDF Report:** إنشاء تقرير رسمي للمراجعة أو التسليم.
- **Visualizations:**
    - Charts → توضيح توزيع الثغرات أو النتائج.
    - Risk Scores → تصنيف الثغرات حسب خطورتها.
    - Vulnerability Cards → بطاقة لكل ثغرة فيها التفاصيل المهمة.
- **Export / Share Results:** مشاركة النتائج مع الفريق أو حفظها للرجوع إليها.

**AI Role:**

- NLP أو LLM لتلخيص النتائج بشكل مفهوم.
- Visualization AI أو مكتبات Python مثل matplotlib / plotly لتوليد الرسوم.

## **Recon & AI-Orchestrated Workflow with AI Roles & Prompts**

---

### **1️⃣ User Input / Target Definition**

**دور الـ AI:**

- **Input Validation:** التأكد من صحة الـ domain/IP/application format.
- **Target Classification:** يحدد نوع الهدف (Web App / Network / API / Cloud).
- **Guided Input:** يقترح للمستخدم خيارات إضافية أو يحذر من بيانات ناقصة.

**مثال برومبت AI:**

```
You are a cybersecurity input assistant.
User provided: <USER_INPUT>
- Verify if the input is a valid domain, IP, or application.
- Suggest corrections or missing fields if needed.
- Classify the input type for the next phase.
Respond with JSON:
{
  "valid": true/false,
  "type": "domain/ip/application",
  "suggested_corrections": []
}

```

---

### **2️⃣ Decision Engine (AI or Rules)**

**دور الـ AI:**

- **Tool Selection:** يختار الأدوات المناسبة بناءً على نوع الهدف.
- **Strategy Optimization:** يحدد أفضل ترتيب للتنفيذ والخيارات لكل أداة.
- **Optional Parameter Tuning:** يحسن إعدادات الأدوات (threads, scan depth, payloads) للحصول على أداء أفضل.

**مثال برومبت AI:**

```
You are a cybersecurity orchestration AI.
Target type: <TARGET_TYPE>
- Recommend a sequence of recon tools for this target.
- For each tool, suggest optimized parameters (threads, ports, scan depth).
- Explain briefly why each tool is selected.
Output JSON:
{
  "tool_sequence": [
    {"tool": "subdomain_enum", "params": {...}},
    {"tool": "port_scan", "params": {...}},
    {"tool": "vuln_scan", "params": {...}}
  ]
}

```

---

### **3️⃣ Execution / Orchestration**

**دور الـ AI:**

- **Orchestrator:** يقرر أي الأدوات تشتغل بالتوازي وأي واحدة تنتظر انتهاء السابقة.
- **Error Handling:** يكتشف الأخطاء أو الفشل في تنفيذ الأداة ويقترح بدائل.
- **Output Normalization:** يحول نتائج الأدوات المختلفة لشكل موحد JSON.

**مثال برومبت AI:**

```
You are an execution assistant AI.
- Receive raw outputs from tools: <RAW_OUTPUTS>
- Normalize and merge all outputs into unified JSON format.
- If errors found in tools, suggest retry or alternative tool.
Output:
{
  "normalized_results": {...},
  "errors": [...],
  "next_steps": [...]
}

```

---

### **4️⃣ Feedback / Adaptive Loop**

**دور الـ AI:**

- **Result Analysis:** تحليل النتائج لاكتشاف نقاط جديدة.
- **Adaptive Planning:** يقرر الخطوة التالية بناءً على النتائج الجديدة.
- **Intelligence Loop:** يضيف الأدوات أو يغير ترتيبها لو ظهرت نتائج غير متوقعة.

**مثال برومبت AI:**

```
You are a cybersecurity adaptive AI.
- Analyze normalized results: <NORMALIZED_JSON>
- Detect new subdomains, open ports, or vulnerabilities.
- Recommend next steps and any additional scans.
Output JSON:
{
  "new_findings": [...],
  "next_plan": [
    {"tool": "http_enum", "params": {...}},
    {"tool": "vuln_scan", "params": {...}}
  ]
}

```

---

### **5️⃣ Reporting / Dashboard**

**دور الـ AI:**

- **Summarization:** تلخيص الثغرات والنتائج المهمة.
- **Risk Scoring:** تعيين درجات خطورة لكل ثغرة.
- **Visualization Guidance:** تحديد أي مخططات/رسوم تبين النتائج بوضوح.

**مثال برومبت AI:**

```
You are a cybersecurity reporting AI.
- Receive full_data.json with all recon results.
- Summarize vulnerabilities, open ports, and findings.
- Assign risk scores (low, medium, high) for each finding.
- Suggest charts or visualizations for dashboard.
Output JSON:
{
  "summary": "...",
  "findings": [...],
  "risk_scores": [...],
  "visualization_recommendations": [...]
}

```
